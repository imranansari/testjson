# C4 Diagram Examples in Mermaid

## 1. Context Diagram

This diagram shows the system in context with its users and other systems it interacts with.

```mermaid
C4Context
title System Context diagram for Internet Banking System

Person(customerA, "Banking Customer", "A customer of the bank, with personal bank accounts")
System(bankSystem, "Internet Banking System", "Allows customers to view information about their bank accounts, and make payments")

System_Ext(mailSystem, "E-mail system", "The internal Microsoft Exchange e-mail system")
System_Ext(mainframe, "Mainframe Banking System", "Stores all of the core banking information about customers, accounts, transactions, etc.")

Rel(customerA, bankSystem, "Uses")
Rel_Back(customerA, mailSystem, "Sends e-mails to")
Rel_Neighbor(bankSystem, mailSystem, "Sends e-mails", "SMTP")
Rel(bankSystem, mainframe, "Uses")
```

## 2. Container Diagram

This diagram zooms in on the system, showing the high-level technical building blocks.

```mermaid
C4Container
title Container diagram for Internet Banking System

Person(customer, "Banking Customer", "A customer of the bank, with personal bank accounts")

System_Boundary(c1, "Internet Banking") {
    Container(web_app, "Web Application", "Java, Spring MVC", "Delivers the static content and the Internet banking SPA")
    Container(spa, "Single-Page App", "JavaScript, Angular", "Provides all the Internet banking functionality to customers via their web browser")
    Container(mobile_app, "Mobile App", "Xamarin", "Provides a limited subset of the Internet banking functionality to customers via their mobile device")
    ContainerDb(database, "Database", "Oracle Database Schema", "Stores user registration information, hashed auth credentials, access logs, etc.")
    Container(api, "API Application", "Java, Spring Boot", "Provides Internet banking functionality via a JSON/HTTPS API")
}

System_Ext(email_system, "E-mail System", "The internal Microsoft Exchange e-mail system")
System_Ext(banking_system, "Mainframe Banking System", "Stores all of the core banking information about customers, accounts, transactions, etc.")

Rel(customer, web_app, "Uses", "HTTPS")
Rel(customer, spa, "Uses", "HTTPS")
Rel(customer, mobile_app, "Uses")

Rel_Neighbor(web_app, spa, "Delivers")
Rel(spa, api, "Uses", "JSON/HTTPS")
Rel(mobile_app, api, "Uses", "JSON/HTTPS")

Rel_Back_Neighbor(database, api, "Reads from and writes to", "JDBC")

Rel(api, email_system, "Sends e-mail using")
Rel(api, banking_system, "Uses", "XML/HTTPS")
```

## 3. Component Diagram

This diagram zooms in on an individual container, showing the components inside it.

```mermaid
C4Component
title Component diagram for Internet Banking System - API Application

Container_Boundary(api, "API Application") {
    Component(sign_in_controller, "Sign In Controller", "MVC Rest Controller", "Allows users to sign in to the Internet Banking System")
    Component(security_component, "Security Component", "Spring Security", "Provides functionality related to signing in, changing passwords, etc.")
    Component(mbti_controller, "Mainframe Banking Transaction Controller", "MVC Rest Controller", "Allows users to interact with banking transactions")
    Component(banking_service, "Banking Service", "Spring Service", "Provides banking transaction functions and interacts with the mainframe")
}

ContainerDb(database, "Database", "Oracle Database Schema", "Stores user registration information, hashed auth credentials, access logs, etc.")
Container(spa, "Single-Page App", "JavaScript, Angular", "Provides all the Internet banking functionality to customers via their web browser")
System_Ext(mainframe, "Mainframe Banking System", "Stores all of the core banking information about customers, accounts, transactions, etc.")

Rel(spa, sign_in_controller, "Uses", "JSON/HTTPS")
Rel(spa, mbti_controller, "Uses", "JSON/HTTPS")

Rel(sign_in_controller, security_component, "Uses")
Rel_Neighbor(sign_in_controller, database, "Reads from and writes to", "JDBC")

Rel(mbti_controller, banking_service, "Uses")
Rel_Neighbor(banking_service, database, "Reads from and writes to", "JDBC")
Rel(banking_service, mainframe, "Uses", "XML/HTTPS")
```

## 4. Code Diagram

This diagram zooms in on an individual component, showing how it is implemented.

```mermaid
C4Code
title Code diagram for Internet Banking System - Security Component

CodeComponent_Boundary(security_component, "Security Component") {
    Class(security_config, "SecurityConfig", "Configures Spring Security")
    Class(jwt_filter, "JwtAuthenticationFilter", "Handles JWT-based authentication")
    Class(user_details_service, "CustomUserDetailsService", "Loads user-specific data")
    Class(jwt_util, "JwtUtil", "Provides JWT operations")
}

Class_Ext(user_repository, "UserRepository", "Spring Data Repository")
Class_Ext(password_encoder, "PasswordEncoder", "BCrypt")

Rel(security_config, jwt_filter, "Configures")
Rel(jwt_filter, jwt_util, "Uses")
Rel(jwt_filter, user_details_service, "Uses")
Rel(user_details_service, user_repository, "Uses")
Rel(security_config, password_encoder, "Configures")
```
